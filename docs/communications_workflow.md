# Communications Orchestration Workflow v1

## Objective

Introduce a communications execution layer that:

1. Receives communication requests from existing application workflows.
2. Sends email automatically.
3. Falls back to SMS where configured.
4. Tracks delivery outcomes.
5. Escalates exceptions to registrars.
6. Returns execution status back to the originating workflow.

The existing member database remains authoritative.

---

# Architecture Principles

- Existing schema remains source of truth.
- No duplicated member/contact records.
- Communication layer owns execution only.
- Provider implementation must be replaceable.
- Prefer Email → SMS → Registrar Queue.
- WhatsApp remains manual registrar fallback.

---

# Communication Request Contract

```json
{
  "event_type": "invoice_due",
  "member_id": 12345,
  "priority": "normal",
  "context": {
    "invoice_id": 556,
    "amount": 1200,
    "due_date": "2026-06-15"
  }
}
```

Rules:

- member_id must exist
- no email supplied
- no phone supplied
- communications layer resolves contacts

---

# Recipient Resolution

Input:

member_id

Lookup existing schema.

Retrieve:

- email
- phone
- status
- preferences

Validation:

- inactive → reject
- missing email → skip email
- missing phone → disable SMS

---

# Template Resolution

Resolve by event type.

Examples:

- invoice_due
- invoice_overdue
- meeting_reminder
- membership_expiry

Store:

- template_id
- variables

---

# Channel Selection Policy

EMAIL
↓
SUCCESS → COMPLETE

FAIL
↓
SMS

FAIL
↓
REGISTRAR QUEUE

Rules:

- No email → SMS
- Email bounced → SMS
- SMS unavailable → Queue
- No response → Queue
- Registrar completed → Close

# Registrar Queue

States:

AVAILABLE
↓
LOCKED
↓
IN_PROGRESS
↓
COMPLETE

Rules:

- lock duration = 15 minutes
- heartbeat = 60 seconds
- first lock wins
- auto release enabled

# Definition of Done

A communication generated by the existing application must:

1. Execute automatically.
2. Record delivery state.
3. Escalate when necessary.
4. Allow registrar intervention.
5. Return final status to the originating workflow.
